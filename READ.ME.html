Il CamaLEOnte - Versione Ibrida (PDF + Video)

Questa documentazione illustra l'architettura tecnica del progetto, passata da immagini statiche a un sistema ibrido Vettoriale (PDF.js) + Multimediale (Veo AI).

1. Architettura del Sistema

  • Rendering Ibrido: Utilizza PDF.js per renderizzare i file PDF su Canvas HTML5 (nitidezza Retina) e Tag HTML <video> per le pagine animate.
  • Layered DOM: Ogni pagina ha due livelli sovrapposti (z-index):
    • Livello 1 (Sotto): Il PDF o il Video.
    • Livello 10 (Sopra): I link interattivi (per evitare sovrascritture).
  • Onboarding Immersivo: Tutorial animato in puro CSS (Glassmorphism) che appare al primo avvio (gestito via localStorage).
  • Responsive Logic: Algoritmo matematico che ridimensiona le pagine senza zoom CSS, prevenendo bug sui dispositivi mobili.

2. Standard dei File (Importante)

Il sistema Ã¨ Case Sensitive. Seguire rigorosamente queste regole:

  • PDF: Nominare 1.pdf, 2.pdf... (tutto minuscolo).
  • Video: Formato .mp4. Nominare come il numero pagina (es. 22.mp4). Devono essere privi di traccia audio (o mutati) per l'autoplay.
  • Numerazione: I file devono essere consecutivi. Non saltare numeri.
