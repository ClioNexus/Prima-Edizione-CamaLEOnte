Documentazione "Il CamaLEOnte"

Versione 1.0 - Febbraio 2026

Questo documento riassume lo sviluppo, l'architettura tecnica e le procedure di manutenzione per la rivista digitale interattiva "Il CamaLEOnte".

1. Panoramica del Progetto

La rivista è un'applicazione web basata su un motore di sfoglio realistico (Turn.js) che serve contenuti originati da Microsoft Word e convertiti in alta risoluzione.

  • Numero Pagine: 24.
  • Articoli Inclusi: Prefazione "Dentro la Selva Oscura", Olimpiadi 2026, Cinema & Mafia, Esperienza TV, Intervista Casati.

2. Stack Tecnologico

Componente Tecnologia/File
Linguaggi HTML5, CSS3, JavaScript (ES6+)
Motore Sfoglio Turn.js (versione 4.1.0)
Libreria Base jQuery (versione 3.6.0)
Hosting GitHub Pages

3. Caratteristiche Implementate

A. Interattività

  • Indice Dinamico (Pagina 3): 12 Hotspot invisibili che reindirizzano alle rispettive sezioni della rivista.
  • Chiusura "Chameleon" (Pagina 23): Un layer di testo pulsante con effetto arcobaleno che, se cliccato, riporta l'utente all'indice.
  • Zoom Engine: Sistema di ridimensionamento fluido che mantiene la proporzione 960x680px su qualsiasi dispositivo.

B. Sicurezza & Anti-Copia

Nota: Le protezioni sono integrate nell'index.html per proteggere il copyright degli autori.
  • Blocco Sorgente: Disabilitazione del tasto destro, F12, Ctrl+U e Ctrl+S.
  • Debugger Trap: Un loop infinito che blocca l'ispezione degli elementi se la console sviluppatore viene aperta.
  • Image Protection: Inibizione del trascinamento (drag & drop) delle immagini PNG.

4. Manutenzione e Aggiornamento

Pipeline di Lavoro

  1. Creare o modificare l'articolo in Microsoft Word.
  2. Esportare in PDF.
  3. Convertire PDF in PNG a 300 DPI (Risoluzione consigliata: 1920x1360px) per garantire nitidezza.
  4. Rinominare il file seguendo la numerazione (es. 5.png) e caricarlo nella cartella principale.

Modifica dei Link (Hotspots)

Per aggiungere un nuovo link, aggiornare l'array map nello script di index.html:

{ 
  t: 20, // Distanza dal top in %
  l: 5,  // Distanza da left in %
  w: 40, // Larghezza in %
  h: 8,  // Altezza in %
  p: 4   // Pagina di destinazione
}

5. Risoluzione Problemi

  • I link non funzionano: Verificare che la delegazione degli eventi ($(document).on('click', ...)) sia attiva nello script per evitare la perdita dei listener durante lo sfoglio.
  • Immagini sgranate: Assicurarsi che i PNG siano stati generati a 300 DPI.
  • Sfoglio lento: Verificare che i file PNG non superino i 2MB l'uno per ottimizzare il caricamento su GitHub Pages.
